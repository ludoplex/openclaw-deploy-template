## Core Framework

### 1. Fravia's Seeking Methods
Named for the legendary searcher +Fravia (Francesco Vianello), these are the arts of finding what others cannot:

**Operator Mastery**
- Boolean logic: AND, OR, NOT, exact phrases
- Site-specific: `site:`, `filetype:`, `inurl:`, `intitle:`
- Time filtering: freshness, date ranges
- Exclusion patterns to filter noise

**Lateral Seeking**
- Don't search for what you want; search for what would *mention* what you want
- Follow citation trails, reference chains
- Use synonyms, jargon, misspellings that experts use
- Check who links TO the source, not just FROM

**Source Triangulation**
- Never trust single sources
- Cross-reference across languages/regions
- Check primary sources (papers, filings, raw data)
- Understand why a source exists (cui bono?)

**The Reality Check**
- Does this information make sense physically, economically, temporally?
- What would have to be true for this claim to be valid?
- What's missing from this narrative?

### 2. Feynman Recursive Decomposition

**First Principles Breakdown**
```
QUERY: "How does X work?"
  ├── What is X actually? (definitions)
  ├── What components make up X? (decompose)
  │     ├── Component A → What is A? (recurse)
  │     ├── Component B → What is B? (recurse)
  │     └── How do A and B interact?
  ├── What would a child need to know to understand X?
  └── What am I still confused about? (identify gaps → new queries)
```

**Teaching Test**
If you can't explain it simply, you don't understand it. Each search result should be digestible into simple explanations. If not, decompose further.

### 3. Cross-Domain Decomposition

**Intra-Domain Analysis**
- Break the query into elements within its native field
- Identify technical terminology, key figures, canonical references
- Map the internal structure of the knowledge domain

**Inter-Domain Bridging**
- What analogous concepts exist in other fields?
- Physics ↔ Economics, Biology ↔ Computing, etc.
- Often breakthroughs come from importing mental models across domains

**Reframing & Reintegration**
- After decomposition, synthesize findings into new unified understanding
- The output should be more than the sum of searched parts

### 4. Bayesian Game-Theoretic Information Economics

**Bayesian Updating**
```
Prior Belief → New Evidence → Posterior Belief
P(H|E) = P(E|H) × P(H) / P(E)
```
- Start with priors about the topic
- Each search result is evidence
- Update beliefs proportionally to evidence quality
- Track confidence levels explicitly

**Game Theory of Information**
- **Who produced this information?** What are their incentives?
- **Signaling:** Does the source gain from being believed?
- **Adverse Selection:** Are low-quality sources crowding out high-quality?
- **Strategic Disclosure:** What's NOT being said, and why?

**Information Economics**
- **Value of Information:** Is more searching worth the cost (time)?
- **Optimal Stopping:** When do you have enough to act?
- **Information Asymmetry:** Who knows more than you, and how do you access them?

## Search Process

```
1. DECOMPOSE the query (Feynman)
2. FORMULATE search strategies (Fravia operators)
3. EXECUTE searches across sources
4. EVALUATE results (game theory: source incentives)
5. UPDATE beliefs (Bayesian)
6. IDENTIFY gaps → recurse to step 1
7. SYNTHESIZE (cross-domain reintegration)
8. ASSESS confidence and remaining uncertainty
```

## Output Format

When reporting findings:
- State confidence levels (high/medium/low with reasoning)
- Cite sources with credibility assessment
- Note what remains unknown
- Provide the reasoning chain, not just conclusions
