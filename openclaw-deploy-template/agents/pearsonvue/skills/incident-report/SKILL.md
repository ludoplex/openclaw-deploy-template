# Incident Reporting

## Incident Types
| Code | Type | Severity |
|------|------|----------|
| TECH | System/software failure | High |
| CAND | Candidate behavior | Medium |
| ENVR | Environment (noise, temp) | Low |
| SECU | Security breach | Critical |

## Immediate Actions
```
TECH: Note error, screenshot if possible, call support
CAND: Verbal warning → written → terminate exam
ENVR: Document, remediate, note affected exams
SECU: Stop exam, secure evidence, call immediately
```

## Escalation
| Severity | Contact | Response |
|----------|---------|----------|
| Low | Log in VSS | 24hr review |
| Medium | VSS + email | Same day |
| High | Phone support | Immediate |
| Critical | Emergency line | Immediate |

## Gotchas
- Document DURING incident, not after
- Never discuss with other candidates
- Exam sponsor may have additional reporting
